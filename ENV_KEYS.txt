
Provision these environment variables (or `tmpkeys.txt` entries) for production deployments:
- STORAGE_MODE: set to `supabase` to enable persistent storage (falls back to `memory` for local tests).
- SUPABASE_URL: the Supabase project URL from Project Settings â†’ API.
- SUPABASE_SERVICE_ROLE_KEY: Supabase service role key for authenticated storage access.
- SUPABASE_STORAGE_BUCKET: bucket name that stores transcripts and manifests.
- SUPABASE_ANON_KEY (optional): anon key for any client-side Supabase usage.
- OPENAI_API_KEY: optional; enables AI-generated follow-ups.
- RESEND_API_KEY: optional; sends the summary email on finalize.
- DEFAULT_NOTIFY_EMAIL: server-side default recipient (must be explicitly configured; placeholders rejected).
- NEXT_PUBLIC_APP_NAME and patience knobs are optional for UI polish.
